# mdocx 日本語動作確認用テストファイル

このファイルは、日本語（UTF-8）のテキストが正常に Markdown と DOCX 間で相互変換されるかを確認するためのテストファイルです。

## 基本的なテキスト装飾のテスト

通常の段落テキストの中に、**太字の日本語テキスト**、*斜体の日本語テキスト*、および~~打ち消し線付きの日本語テキスト~~が混在している場合の動作を確認します。また、インラインでの `コード表示` も正しく変換される必要があります。

## リスト構造のテスト

### 箇条書き（無順序リスト）
* 日本語のリスト項目1
* 日本語のリスト項目2
  * ネストされたレベル2の項目A
  * ネストされたレベル2の項目B
* 日本語のリスト項目3

### 番号付きリスト（順序リスト）
1. 最初のステップ
2. 次のステップ
   1. サブステップその一
   2. サブステップその二
3. 最後のステップ

## テーブルのテスト

## 引用とコードブロック

> これは日本語の引用ブロック（Blockquote）です。
> 複数行にわたる引用や、段落全体のインデントと文字スタイルが DOCX 側でも保持される必要があります。

以下は Rust のコードブロックの例です：

```rust
fn main() {
    let message = "こんにちは、mdocx！";
    println!("{}", message);
}
```
